Профиль настроек Horae
Экспорт всех переключателей, настроек «только ГГ», пользовательских промптов, RPG-конфигурации в файл (без авто-сводки и векторной памяти). Авторы карт могут делиться конфигурацией вместе с картой, игроки импортируют и всё готово.
Основные настройки
При отключении иконка полностью скрывается. Можно снова включить через панель расширений (иконка «кубики») в разделе «Horae — Память Времени».
Процент ширины панели сообщений от области чата (50-100). Для широкоэкранных тем можно уменьшить. Смещение позволяет подстроить горизонтальное положение панели.
Внешний вид
Выберите встроенную тему или импортированную пользовательскую. После импорта файла темы (.json) она появится в выпадающем списке.
Пользовательский CSS
Введите пользовательский CSS для переопределения стилей плагина. Подходит для продвинутых пользователей. Изменения применяются при потере фокуса.
Расширенные настройки
Количество «обычных» событий, отправляемых AI:
0 = не отправлять обычные события (важные/ключевые всё равно отправляются)
15 = отправлять последние 15 обычных событий (по умолчанию)
Чем больше число, тем больше токенов потребляется
В какую позицию промпта AI вставляется информация Horae:
0 = в самый конец (рядом с последним сообщением)
1 = предпоследняя позиция (рекомендуется, даёт AI пространство для размышлений)
Чем больше число, тем ближе к началу
Пользовательские таблицы
Создавайте таблицы в стиле Excel, AI заполнит их по вашим требованиям.
Долгое нажатие на заголовок позволяет добавлять/удалять строки и столбцы. Для каждой таблицы можно настроить промпт, указывающий AI, как заполнять.
Локальные таблицы сохраняются с текущим диалогом; Глобальные таблицы общие для всех диалогов. Кликните иконку перед названием для переключения.
🔒 Блокировка строк/столбцов предотвращает их изменение AI.
Отправка данных AI
Управление тем, какая информация отправляется AI вместе с промптом. Отключение экономит токены, но AI не сможет «помнить» эту информацию.
Текущие внедрённые токены：Подсчёт...
Сводки исторических событий, расчёт относительного времени. При отключении AI не узнает, что произошло «несколько дней назад».
Присутствующие персонажи, описание костюмов, симпатия, данные персонажей. При отключении AI может описывать неправильные костюмы.
Текущие предметы и их местоположение. При отключении AI может «достать из воздуха» несуществующие предметы.
Фиксирует физические особенности мест для согласованности описаний между ходами. При включении AI записывает описание при первом посещении нового места.
Отслеживание изменений отношений между персонажами (друзья, любовники, враги и т.д.), AI выводит только при изменениях. При отключении не генерируются теги rel:, не внедряется промпт сети отношений.
Отслеживание эмоций/психического состояния присутствующих персонажей для согласованности. AI выводит только при заметных изменениях эмоций.
При включении AI будет выводить шкалы атрибутов (HP/MP/SP) и изменения навыков в теге <horaerpg>. При отключении промпт не внедряется, токены не тратятся.
Для стиля, когда USER сам описывает действия, а AI не пересказывает часть USER. При включении AI при ответе также включит содержимое предыдущего сообщения USER в расчёт тегов <horae>, чтобы действия USER не были пропущены.
При включении в нижней панели появится кнопка «Пометить как спин-офф». Помеченные сообщения не отслеживаются Horae и не участвуют в авто-сводке, подходит для вставки if/спин-оффов/мини-сценок в основной сюжет.
Авто-сводка и скрытие
При превышении лимита буфера старые события автоматически сжимаются AI в сводки, экономя токены.
Сжатые сводки можно развернуть обратно в исходные события на таймлайне.
Последние N сообщений никогда не будут сжаты. Обеспечивает полное сохранение деталей недавнего сюжета.
При превышении этого лимита запускается автоматическое сжатие.
Пример: допустим, сохранять=10, лимит буфера=20 (режим сообщений)
→ Когда вы дойдёте до 50-го сообщения: сообщения 41-50 безусловно сохраняются, не сжимаются
→ Из сообщений 1-40, если несжатых более 20, запускается авто-сводка
→ Каждый раз обрабатывается пакет по настройкам ниже (кол-во сообщений или токенов), остаток — в следующий раз
Просто: сохранять + буфер = после скольких сообщений начинается сводка
Максимальное количество сообщений за один цикл авто-сводки. При первом включении для старых архивов обработка идёт в несколько циклов.
Лимит токенов текста за одну сводку, при превышении обрезается до этого значения.
Ориентир по контекстному окну моделей (рекомендуется 30%~50%):
Claude ≈ 80K~200K · GPT-4o ≈ 80K~128K · Gemini ≈ 100K~1000K · DeepSeek ≈ 64K
Укажите OpenAI-совместимый эндпоинт — сводки будут генерироваться напрямую через него, не занимая основное подключение SillyTavern.
Базовый адрес OpenAI-совместимого эндпоинта, например https://api.openai.com/v1
Заполните адрес и ключ, затем нажмите для загрузки. Подойдёт недорогая модель.
После заполнения авто-сводки будут генерироваться через отдельный HTTP-запрос параллельно, не затрагивая основной API SillyTavern.
Поддерживаются все OpenAI-совместимые эндпоинты (OpenAI, DeepSeek, OpenRouter и др.).
Тест подключения только запрашивает список моделей и не расходует генерации.
Векторная память (Beta)
Векторный поиск по данным тегов Horae, автоматический вызов релевантных исторических фрагментов для AI.
Локальный режим: скачивание ~25MB квантованной модели в кэш браузера, потребление 50-80MB памяти. ⚠ Не рекомендуется для телефонов с менее 2GB ОЗУ или облачных SillyTavern — обязательно настройте --max-old-space-size для снятия лимита памяти Node.js.
API-режим: использование удалённого Embedding API, без нагрузки на локальные ресурсы.
q4 подходит для большинства устройств; q8 точнее, но вдвое больше по размеру.
Указывайте только Embedding-модели (модели эмбеддингов/векторов)!
Не указывайте LLM-модели для чата (например gemini 3, gpt-4 и т.д.), иначе будет ошибка.
Рекомендуется: text-embedding-v3 (Qwen), text-embedding-3-small (OpenAI)
OpenAI-совместимый формат, путь /embeddings добавляется автоматически.
Пример: https://dashscope.aliyuncs.com/compatible-mode/v1
Если провайдер использует нестандартный путь — укажите полный URL сюда. Например: https://xexexexe.sbs/v1/embeddings
Заполните адрес и ключ, затем нажмите для загрузки.
Рекомендуется: text-embedding-v3 (Qwen) / text-embedding-3-small (OpenAI)
Подходит для высокоразмерных API-моделей (Qwen3-Embedding, text-embedding-3-large и др.).
При включении отключаются расширение ключевых слов, сопоставление фраз событий, штраф высокочастотных слов и другие вспомогательные эвристики,
полностью полагаясь на семантическое понимание модели для вызова, снижая низкокачественный шум.
Если вы используете малую или низкоразмерную модель, не включайте — качество вызова снизится.
После векторного вызова Rerank-модель выполняет точную сортировку кандидатов, значительно повышая точность вызова:
действительно релевантное поднимается вверх, менее релевантное опускается вниз — AI получает более точные воспоминания.
Требуется дополнительная настройка API Rerank-модели (отличается от Embedding-модели).
Рекомендуемые модели: Qwen3-Reranker, bge-reranker-v2-m3, bce-reranker
Заполните адрес и ключ, затем нажмите для загрузки.
Рекомендуется: gte-rerank (Qwen) / jina-reranker-v2-base-multilingual (Jina)
По умолчанию Rerank использует сводки для сортировки. При включении используется полный текст исходного сообщения для точной сортировки,
что позволяет уловить детали, пропущенные в сводках, но предъявляет высокие требования к контекстному окну модели.
Только для Rerank-моделей с длинным контекстом (например Qwen3-Reranker, поддержка 32K).
Для моделей с коротким контекстом: bge-reranker-v2-m3 (≤1024) и bce-reranker (≤512)
не включайте — полный текст будет обрезан, и результат станет хуже.
При каждой отправке сообщения из векторной базы вызываются K наиболее релевантных воспоминаний для внедрения в контекст.
Воспоминания со сходством ниже этого значения не будут вызваны. Повышение уменьшает шум.
Топ-N результатов, достигших полнотекстового порога, отправляются как полный текст, а не сводка. Установите 0 для отключения.
Только результаты с высокой достоверностью (сходство ≥ этого значения) будут отправлены полным текстом, чтобы нерелевантный контент не тратил токены.
Укажите имена тегов через запятую. В следующих сценариях контент внутри этих тегов будет полностью удалён:
· Векторный поиск — полнотекстовый вызов и Rerank по полному тексту
· Парсинг таймлайна — предотвращает загрязнение таймлайна тегами horae из мини-сценок
· AI-сводка — удаляется перед отправкой сообщений AI для анализа
Пример: snow, theater удалит <snow>...</snow> и <theater>...</theater>.
HTML-комментарии <!-- ... --> удаляются по умолчанию, указывать не нужно.
